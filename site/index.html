chapter bookmarks for PDFs that don’t have any.

walnut reads a PDF, figures out where the chapters start, and writes a proper outline back into a copy of the file. it runs entirely on your machine.

see how it works →

free, open source, no account, no cloud.

how it works

  1. 01

    drop a PDF in your browser

    walnut runs as a tiny local web app. open http://localhost, drag any PDF in — book, paper, scanned report — and it starts reading.

  2. 02

    detect chapters with a local model

    if the PDF has a printed Table of Contents, walnut parses it directly. otherwise it asks Gemma 4, running locally via Ollama, to infer chapter boundaries from the text and font sizes.

  3. 03

    review, then save

    before anything is written, walnut shows you the detected chapters. rename, nudge a page number, delete a bogus one, or accept and download. the output is byte-identical to your input apart from the appended outline.

why local

your PDFs never leave your machine.

no upload to a server, no third-party API, no analytics. walnut spawns a Python process bound to 127.0.0.1. that’s the entire surface area.

built on Ollama and Gemma 4.

Gemma 4’s 4B effective-parameter model fits in 16 GB of RAM and runs at 18–24 tokens per second on Apple Silicon via the MLX backend.

byte-equivalent output.

the saved file is the original input plus an appended PDF incremental update. signatures stay valid; nothing else in the file changes.

open source.

read the source, file an issue, send a patch. MIT licensed code on GitHub.

install

walnut needs Python 3.12+ and an Ollama daemon. on macOS:

brew install ollama
brew services start ollama
ollama pull gemma4:e4b

uvx walnut

first run pulls a ~9.6 GB model. after that it’s instant.

about

walnut is a one-person side project. it exists because a lot of book PDFs ship without an outline, and reading a 600-page novel without bookmarks is no fun.
