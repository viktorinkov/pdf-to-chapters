    how it works

    1. 01

      drop a PDF in your browser

      walnut runs entirely on this page. drag any PDF onto the dropzone — book, paper, scanned report — and it starts reading. nothing leaves your machine.

    2. 02

      detect chapters

      walnut looks for a printed Table of Contents first. if there isn’t one, it falls back to font-size heuristics. for harder PDFs, you can spin up a local LLM on demand — it loads into your browser via WebGPU, no server.

    3. 03

      review, then save

      before anything is written, walnut shows you the detected chapters. rename, nudge a page number, delete a bogus one, or just hit save. the download is your input PDF with a proper outline added.

    why local

    your PDF never leaves your tab.

    this page is a static site on GitHub Pages. there is no server to upload to, no API to call, no log of your file ever existing. the PDF stays in your browser’s memory and is dropped when you close the tab.

    two model options.

    the browser version uses Gemma 2 (~1.5 GB) loaded into your tab via WebGPU when you ask for it. the desktop version uses Gemma 4 via Ollama for harder books.

    byte-stable output.

    the saved file is your original PDF with an outline section appended — the bytes you started with are preserved exactly, with new objects, an xref, and a trailer added on the end. page contents, fonts, and page tree are untouched. big books work the same as small ones.

    open source.

    read the source, file an issue, send a patch. MIT licensed code on GitHub. no analytics, no third-party scripts, no cookies.

    what’s under the hood

    walnut detects chapters in three passes: a regex over a printed Table of Contents page (dot leaders + right-aligned page numbers), a font-size heading heuristic for books without one, and an opt-in local AI pass for the rest. all three run in your browser; nothing is uploaded.

    • PDF.js 5.6 — Mozilla’s PDF parser. reads per-page text plus per-line font size and position so the heuristic can spot heading-sized lines.
    • pdf-lib 1.17 — fallback writer for PDFs whose catalog is hidden inside a compressed object stream. the default path is a hand-rolled PDF incremental update so the output is byte-identical to the input plus an appended outline section.
    • WebLLM + Gemma 2 (2B, 4-bit) — runs in your browser via WebGPU when neither the TOC parser nor the font heuristic finds anything. ~1.5 GB one-time download cached locally; no API key, no server. the desktop version uses Gemma 4 (E4B) via Ollama for higher quality.

    prefer the command line?

    there’s a desktop version that runs Gemma 4 locally via Ollama, with the same byte-stable incremental writer used here. on macOS:

    brew install ollama
brew services start ollama
ollama pull gemma4:e4b

uvx --from git+https://github.com/viktorinkov/pdf-to-chapters walnut

    first run pulls a ~9.6 GB model. after that it’s instant. the package isn’t on PyPI yet, so install straight from git with uvx --from git+….

    about

    walnut is a one-person side project. it exists because a lot of book PDFs ship without an outline, and reading a 600-page novel without bookmarks is no fun.
